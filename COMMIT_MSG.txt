SOUL §3 amendment: RESOURCE BOUNDARIES ARE SOCKETS, NOT LINKED LIBRARIES.
Privileged resources (DB, pain, locks) are mediated by socket-fronted services.
Pure-logic code (paths, parsing) stays a library. This section hash changes;
re-baseline at S1.5.

db (client library + probe, no libpq):
  - Sends parameterized SQL over AF_UNIX socket to db-proxy
  - cJSON for response parsing, snprintf for request building
  - Zero DB credentials in client; no libpq link
  - Static lib + header for downstream (pain, locks, etc.)
  - Probe tool (db-probe): connects through socket, SELECT 1 → JSON

db-proxy (daemon, sole libpq owner):
  - Listens on {scratch}/db-proxy.sock or $VEHIR_DB_PROXY_SOCK
  - Gate: blocks DROP, TRUNCATE, ALTER, information_schema, pg_catalog
  - Poll-based event loop, one JSON request per connection (stateless)
  - Connection string from $VEHIR_PG_CONN / $DATABASE_URL / broker env
  - PQexecParams for all queries (bind-safe)
